# 番茄人生 — 产品需求文档

版本 2.0 | 2026年7月3日

> **定位转变**：v1.0 以"AI 角色养成"为核心，v2.0 转向**"GTD 思维框架 × 番茄钟执行 × Obsidian 数据闭环"**——先想清楚，再开始做，做完的数据能被自己管理。

---

## 一、产品概述

### 1.1 一句话定义

一个以 GTD 认知流程为引导的番茄钟工具，帮助知识工作者在专注前"想清楚要做什么"，通过 Obsidian 导出实现数据可管理，为 AI Agent 提供结构化日志。

### 1.2 核心用户

- 需要深度工作的知识工作者（程序员、研究者、写作者）
- 有工具链意识，使用 Obsidian/Notion 做知识管理
- 对 GTD 方法论有认知或有需求，而不是简单地"列待办清单"

### 1.3 解决的痛点

**旧问题**：传统番茄钟用户打开 APP → 看到空任务列表（焦虑）→ 堆任务（更焦虑）→ 盲目开始 → 计时结束做了很多无效努力。

**解决方案**：在"想干活"和"开始干活"之间，用 GTD 四列看板强制插入一个认知加工停顿——必须把模糊念头加工成具体的"下一步行动"再执行。

---

## 二、核心功能

### 2.1 GTD 思维框架（四列看板）

```
CAPTURE  →  CLARIFY  →  NEXT ACTIONS  →  DOING
  收集         加工        下一步行动        执行中
```

- **CAPTURE**：所有念头、任务、想法的倾倒口——零筛选，不掉任何东西
- **CLARIFY**：逐条加工——"需要行动吗？" → NO（丢弃/存档/将来） / YES（下一步是什么？）
- **NEXT ACTIONS**：加工完毕的明确行动项——只有这里能进入 DOING
- **DOING**：当前正在执行的任务——与番茄钟计时器联动

### 2.2 番茄钟计时器

- 支持 25/5、52/17、90/25、Custom 四种节奏
- Focus/Break 双层标签切换
- 每完成一个 focus 自动记录 session（开始/结束时间、时长、关联任务）
- 点阵可视化当日番茄完成数（最多 30 个）

### 2.3 传统任务清单

保留 pomofocus.io 风格的任务列表（Tasks 模式），作为 GTD 模式外的轻量选择。

### 2.4 数据闭环

- **导出 .md**：结构化 Markdown（含 Agent 解析指令、Summary、Sessions、Tasks、GTD Board）
- **导入 .md**：从 Obsidian 编辑后回填到计时器
- **设计目标**：AI Agent 可直接读取日志分析专注模式，推荐优化建议

---

## 三、架构

```
用户工作流：
  打开 → GTD 加工（想清楚）→ 选 DOING 条目 → 番茄钟执行
       ↓
  专注结束 → Session 记录入库
       ↓
  .md 导出 → Obsidian 复盘 → Agent 分析 → 导入回计时器
```

技术栈：
- **产品计时器**：`product/pomofocus.html`（单文件 HTML，GTD + 番茄钟交互原型）
- **后端**：Fastify + Drizzle + PostgreSQL（pgvector）
- **LLM**：DeepSeek `deepseek-chat`
- **客户端**：Expo React Native（待开发）
